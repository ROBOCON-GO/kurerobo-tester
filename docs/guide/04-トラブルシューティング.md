# 困ったとき(トラブルシューティング)

実際に起きた症状と対処をまとめています。

## ページが開けない / localhost が表示されない

- **サーバーが起動しているか**:`start.bat` を実行(黒い画面が出たまま=起動中)。
- `http://localhost:8080` で開く(`https` ではない / `file://` で直接開くのは不可)。
- **Chrome か Edge** で開く(Firefox/Safari は非対応)。

## ブラウザが固まる(操作後しばらくして)

- 古いコードがキャッシュされている可能性。**Ctrl + Shift + R**(ハードリロード)。
- 確認:**F12 → Console** に `KureRobo Tester build: heartbeat-v2 …` が出ていれば最新版。
- 出ていなければ、F12 の Network タブで「Disable cache」にチェックして再読み込み。
- ③操縦で固まる場合、`control()` に**重い処理や無限ループ**を書いていないか確認。

## Bluetooth で接続できない

- Windowsの Bluetooth 設定で `ESP32_CTRL` が**ペアリング済み**か確認。
- 「接続」で選ぶのは **Bluetooth の仮想COM**(USB書き込み用COMではない)。2つあれば**大きい番号**を試す。
- ESP32 を**書き込み/リセットすると一旦BTが切れます**。アプリで「切断→接続」し直す。

## モーターが動かない / UART が出ない

- ②手動テストで、動かす行の**チェックを入れてスライダーを動かしたか**(チェックなしは送られない)。
- 「送信モニタ」の**送信レートが 0 より大きい**か(0なら未接続か未有効化)。
- **Bluetooth側のCOM**に繋いでいるか(USB側だとメインボードに届かない)。
- **新モタドラだけ動かない**場合:
  - モタドラの**CAN ID をボタンで設定**したか(押した回数=ID)
  - 制御5V と モータ電源(7.4〜25.2V)が入っているか
  - CAN H/L が同じバスに繋がり、必要なら**終端抵抗**を有効化
  - duty は **0 だとブレーキ**。300以上にする

## 書き込み後、ESP32 が数秒ごとに再起動する(起動ループ)

- 症状:シリアルに `invalid header` / `RTCWDT_RTC_RESET` が連発。フラッシュ破損です。
- 対処:**`flash.bat`(ポータブル版)で焼き直す**と確実に直ります(Hash検証付き)。
- 原因の確認方法(任意):
  ```
  arduino-cli monitor -p COM4 -c baudrate=115200
  ```
  で起動ログを見ると、リセット理由が分かります。

## 「PCが保護されました」と出て flash.bat / esptool が動かない

- 「**詳細情報 → 実行**」で進めてOK(同梱 esptool.exe が署名なしのため)。

## それでも解決しないとき

- F12 → Console の**赤いエラー文**を控えておくと原因特定が早いです。
- ③操縦なら、編集中の `control()` の内容も合わせて確認してください。
