# ② 手動テストの使い方

スライダーやトグルでモーターを直接動かせます。**プログラムは一切不要**。
機構が完成したら、まずここで単体テストするのがおすすめです。

## 準備:Bluetooth 接続

1. ESP32 に電源を入れる(USB給電でOK)
2. **Windowsの Bluetooth 設定**で `ESP32_CTRL` を**ペアリング**(初回のみ)
3. アプリ右上の「**接続**」を押す → ポート一覧から **Bluetooth の仮想COM** を選ぶ
   - USB書き込み用のCOM(例:CP210x)ではなく、**Bluetooth リンク経由の標準シリアル**の方
   - 2つ出る場合は**大きい番号**を試す(ダメならもう片方)
4. 右上のランプが緑(接続中)になればOK

> 接続中は **50ms ごとに送信し続けます**。これはメインボードが「800ms 通信が途切れると安全停止する」
> 仕様への対策です。

## 各セクションの使い方

行の左の**チェックを入れて有効化** → スライダーを動かすと、その値が送られます。
(チェックを入れていない行は送られません)

| セクション | 内容 |
| --- | --- |
| ロボマスモーター 速度制御 | M2006 / M3508 を速度(rpm)で回す(ID1〜8) |
| GM6020 速度制御 | GM6020 を速度で回す(ID1〜7) |
| 新モタドラ | CAN直結ブラシ付きモタドラ(ID1〜8)。duty -999〜999、**0=ブレーキ** |
| PWM出力 | サーボ/ESC(ch1〜6)。パルス幅 0〜20000us |
| GPIO出力 | GPIOのON/OFFをビットで |

### PWM の「絶縁反転」
現行のフォトカプラ絶縁基板は論理が反転します。**「絶縁反転」にチェック**(既定ON)が入っていると、
スライダーに「**出したいパルス幅**」を入れるだけで、内部で `20000 − 値` に変換して送ります。
絶縁していない基板を使うときはチェックを外してください。

### 新モタドラの「原点リセット」
各行の「原点リセット」ボタンで、ロリコン(ロータリーエンコーダ)のカウンタを0にできます。

## 全停止

右上の「**■ 全停止**」を押すと送信を止めます(モーターは停止)。もう一度押すと再開します。
非常時はこれ、もしくはブラウザのタブを閉じればすぐ止まります。

## 送信モニタ

画面下に「送信レート(回/秒)」「最終フレーム(実際に送ったバイト列)」が出ます。
**送信レートが 0 のまま**なら、行が有効化されていないか、接続できていません。
