# ① ESP32 への書き込み

ESP32 に「ブリッジファーム」を書き込みます。**各ESP32につき最初の一度だけ**でOK(以後は書き換え不要)。

## いちばん簡単な方法:ポータブル書き込みツール(インストール不要)

1. アプリの **①書き込みタブ** の「**⬇ 書き込みツール(ZIP)をダウンロード**」を押す
   - (または `flash-tool.zip` を直接入手)
2. ZIPを**解凍**(右クリック →「すべて展開」)
3. **USBで ESP32 をPCに接続**する
4. 解凍したフォルダの中の **`flash.bat` をダブルクリック**
5. 黒い画面に次が出れば成功:
   ```
   OK! Flash complete (hash verified).
   ```

COMポートは自動検出されます。書き込み中はUSBを抜かないでください。

> **Windowsの警告が出たら**: 「Windows によって PC が保護されました」と出ることがあります
> (同梱の esptool.exe が署名なしのため)。「**詳細情報 → 実行**」で進めてOKです。

## うまくいかないとき

- **「ポートが見つからない」**: USBケーブルがデータ通信対応か確認。デバイスマネージャーに
  `Silicon Labs CP210x (COMx)` が出ているか確認。出ない場合はCP210xドライバを入れる。
- **途中で失敗する**: USBを抜き差ししてもう一度 `flash.bat`。

## なぜブラウザから直接書き込まないの?

ブラウザ書き込み(esptool-js)も試しましたが、このボードのUSBシリアル(CP2102)と相性が悪く、
**書き込み中にフラッシュを破損**させてしまいました(起動ループになる)。
中身が同じ esptool でも exe版/Python版は問題なく書けるため、確実な `flash.bat` 方式にしています。

## (制御班向け)ファームを変更したとき

ブリッジファーム本体は `firmware/bridge/bridge.ino`(Bluetooth↔UART を素通しするだけ)。
**通常は変更不要**です。もし変更したら `arduino-cli` で再ビルドが必要:

```powershell
# winget install ArduinoSA.CLI （初回のみ。ESP32コアは自動取得)
.\flash.bat   （ルートにある方。compile→upload→Hash検証)
```

再ビルド後は `firmware/build/*.bin` を `flash-tool/` にコピーし、`flash-tool.zip` を作り直します。
